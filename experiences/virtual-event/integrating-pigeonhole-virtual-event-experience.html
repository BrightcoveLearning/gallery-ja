Pigeonholeとバーチャルイベントエクスペリエンスの統合 Pigeonholeとバーチャルイベントエクスペリエンスの統合

{{ page.description }}

Pigeonhole を使用すると、バーチャルイベントエクスペリエンスの1つ以上の動画にイベントベースのインタラクティビティ（ライブQ&A、投票、アンケートなど）を簡単に追加できます。通常、ライブイベント用に新しい鳩の巣が作成され、鳩の巣機能を追加するビデオごとにセッションを作成できます。セッションが作成されると、ビデオクラウドのカスタムフィールドを使用して、iframe 埋め込みコードが各動画に関連付けられます。埋め込みコードにより、選択したビデオのビデオプレーヤーの横にPigeonholeポッドが表示されます。

新しいピジョンホールイベントとセッションの作成

ライブイベント用に新しい Pigeonhole を作成し、Pigeonhole 機能が追加されるビデオごとにセッションを追加する必要があります。

  1. Pigeonholeアカウントにログインします。
  2. + Pigeonholeを追加]をクリックし、使用するアカウントをクリックします。
  3. イベント名とパスコードを入力します。
  4. イベントの開始日と終了日を入力します。これらの日付は、ピジョンホールがアクティブになる時間を制御します。
  5. タイムゾーンとカスタムブランディングオプションを選択します (該当する場合)。
  6. [続行]をクリックします。
  7. イベントが作成されたので、左側のナビゲーションでアジェンダアイテムが選択されます。[ セッションを追加] をクリックし、[ 新規作成] をクリックします。
  8. を選択セッションタイプのQ＆A（他のセッションタイプも使用できます）。
  9. セッション名を入力します。
  10. 開始日と終了日時を入力します。
  11. 必要に応じて、 Q&A設定を構成します。
  12. [ Q&Aを追加] をクリックして、セッションが作成されたことを確認します。
  13. 必要に応じて他のセッションを作成します。

Pigeonhole 埋め込みコードを生成する

動画の横に Pigeonhole を表示するには、Video Cloud カスタムフィールドを使用して iframe 埋め込みコードが動画に追加されます。コードの形式は次のようになります。

<iframe src = "鳩の穴のURL「>  <  /iframe >

pigeonhole URL の値を取得するには、次の手順を実行します。

  1. ピジョンホールで、をクリックしますイベントを実行する左側のナビゲーションで。
  2. [ 出席者を招待]セクションのリンクをクリックします。
  3. すべてのセッションのリストを含む新しいブラウザタブが開くことを確認します。
  4. 該当するセッションの [Enter] リンクをクリックします。たとえば、 [このQ&Aを入力してください] をクリックします。リンクはセッションタイプに基づいて変更されることに注意してください。
  5. Pigeonhole が新しいブラウザタブで開くことを確認します。Pigeonhole URL はブラウザのアドレスバーに表示されます。

Pigeonhole URL が付いた iframeタグが適切な動画に追加されます。

イベント設定の構成

Pigeonhole イベントの設定も、SSO 情報を使用してユーザプロファイルを作成できるように設定する必要があります。

  1. 左側のナビゲーションで、[ その他の設定] をクリックします。
  2. [ 出席者 ] タブをクリックします。
  3. [ 埋め込みによる出席者プロファイルの作成を許可する ] オプションをオンにします。
  4. ページの [ 登録者プロファイル ] セクションまでスクロールします。
  5. [ このイベントの登録者プロファイルを有効にする ] オプションをオンにします。

カスタムフィールドを作成する

バーチャルイベントテンプレートがビデオで表示する Pigeonhole を認識するために、カスタムフィールドが使用されます。カスタムフィールドを作成するには、次の手順に従います。

  1. ビデオクラウドスタジオにログインします。
  2. ナビゲーションヘッダーで、[ 管理 ]、[ ビデオフィールド] の順にクリックします。管理者メニューにアクセスするには、アカウント管理者である必要があります。
  3. Chat Embed という名前の新しいカスタムフィールドを作成します。
  4. 内部名が chat_embed であることを確認します。
  5. [ タイプ]を [ テキスト] に設定します。
  6. カスタムフィールドを保存します。

Video Cloud でカスタムフィールドを作成する方法については、カスタムメタデータフィールドの作成を参照してください。

動画に鳩穴を追加する

バーチャルイベントテンプレートのビデオの横に Pigeonhole を表示するには、次の手順に従います。

  1. Media モジュールを開きます。
  2. ビデオをクリックして、ビデオのプロパティを開きます。
  3. [ カスタムフィールド]セクションを見つけて、[ 編集] をクリックします。
  4. Pigeonhole 埋め込みコードをチャット埋め込みカスタムフィールドに貼り付けます。
  5. [保存]をクリックします。

ユーザーのSSO情報をPigeonholeに渡す

仮想イベントエクスペリエンスが SSO を使用してセキュリティで保護されている場合、Pigeonhole 内でユーザーを追跡できるように Pigeonhole にユーザー SSO 情報を渡すことができます。以下のセクションでは、SSO 情報を Pigeonhole に渡すために必要な設定と手順について説明します。

SSO 属性のマッピング

適切な SSO 属性がマッピングされていることを確認して、エクスペリエンスに渡してから Pigeonhole に渡す必要があります。積分Pigeonholeでは、次の値が検索されます。

  • fname -ユーザー名
  • nameID -ユーザーのメールアドレス

以下の例は、これらの属性がOktaSSO管理ページ内から公開されていることを示しています。

アクセス制御プロファイルで属性を公開する

エクスペリエンスの保護に使用されるアクセス制御プロファイルで、[ すべての属性の公開]オプションがオンになっていることを確認します。SSO情報が利用できない場合、ユーザーはPigeonholeで匿名として表示されます。

  1. ギャラリーモジュールを開きます。
  2. クリック設定ヘッダーに。
  3. 左側のナビゲーションで、[ アクセス制御プロファイル] をクリックします。
  4. エクスペリエンスを保護するために使用されているプロファイルの編集アイコンをクリックします。
  5. [ すべての属性を公開 ] 設定がオンになっていることを確認します。

シングルサインオンの構成の詳細については、アクセス制御プロファイルの設定を参照してください。すべてが正しく構成されていれば、ビューアーがエクスペリエンスにログインしたときにユーザー情報が Pigeonhole に表示されます。
