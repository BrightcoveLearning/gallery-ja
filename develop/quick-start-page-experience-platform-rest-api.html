「クイックスタート:ページ内エクスペリエンスプラットフォーム (REST) API

クイックスタートでは、ページ内エクスペリエンスを管理するための API リクエストの例と、それらを使用するためのサンプルインターフェイスを提供します。

はじめに

ページ内エクスペリエンス API を使用すると、ページ内エクスペリエンスをプログラムで管理できるため、CMS 統合に特に役立ちます。これにより、ユーザーは Studio の [ギャラリー] に移動するのではなく、自分の CMS から管理エクスペリエンスを作成することができます。

このクイックスタートでは、ユーザーは次のことを可能にする実装例を提供します。

  1. アカウントのすべてのページ内エクスペリエンステンプレートを取得する
  2. 選択したテンプレートから基本的なエクスペリエンスを作成する
  3. プレイリストまたは動画のリストを選択して、エクスペリエンスに追加します
  4. 選択した動画/プレイリストでエクスペリエンスを更新する
  5. エクスペリエンスを公開する

以下に、アプリロジックの概要を示します。

プロキシコード

このページのサンプルアプリを独自のバージョンで作成するには、独自のプロキシを作成してホストする必要があります。（ブライトコーブラーニングサービスで使用されるプロキシは、ブライトコーブのドメインからのリクエストのみを受け付けます）。プロキシコードの 2 つのバージョンをダウンロードできます。

  • これは、クライアント資格情報が要求とともに渡されることを期待する一般的なバージョンです
  • このバージョンでは、25-26 行目のプロキシ自体にクライアントの資格情報を保存できます (推奨)
